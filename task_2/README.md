# Task 2: Tabular Regression

## Overview

The dataset contains 53 anonymized features (`0`вЂ“`52`) and a `target` column. The goal is to
predict `target` for `hidden_test.csv`, evaluated with RMSE.

EDA revealed that the target is generated by an exact, deterministic formula:

```
target = column_6 ** 2 + column_7
```

This relationship holds across every row of `train.csv` to floating-point precision (max
absolute error в‰€ 5.7e-14), and the same value ranges for columns `6` and `7` are present in
`hidden_test.csv`, consistent with the same generative process. The other 51 features behave like
independent noise: roughly symmetric distributions, no meaningful linear or rank correlation with
the target, and no residual signal once columns `6` and `7` are accounted for.

The final model is **not** a hardcoded formula. It is a small, properly cross-validated
`scikit-learn` pipeline (`PolynomialFeatures(degree=2)` + `LinearRegression`, trained on columns
`6` and `7`) that *independently learns* this relationship from the data. The fitted coefficients
confirm the discovery: the model assigns a coefficient of в‰€1.0 to `column_6ВІ`, в‰€1.0 to `column_7`,
в‰€0 to every other polynomial term, and an intercept of в‰€0.

## Running the EDA notebook

```bash
jupyter notebook notebooks/eda.ipynb
```

The notebook walks through the discovery process step by step: a broad linear-correlation scan
across all 53 features (which shows essentially nothing), a nonlinear feature-importance probe
that isolates column `6` as the dominant feature, a scatter plot confirming the quadratic shape,
and a residual check that identifies column `7` as the remaining term.

## Training the model

```bash
python train.py --train-path data/train.csv --model-path artifacts/model.joblib
```

Both arguments have sensible defaults (`data/train.csv`, `artifacts/model.joblib`), so the script
also runs with no arguments from inside `task_2_regression/`.

`train.py`:
1. loads `train.csv` and validates that columns `6`, `7`, and `target` are present with no
   missing values;
2. evaluates the model with 5-fold cross-validation (`KFold(shuffle=True, random_state=42)`),
   printing per-fold and mean/std RMSE;
3. refits the model on the full training set;
4. saves a bundle containing the fitted model and the exact list of feature columns it expects to
   `artifacts/model.joblib`.

## Generating predictions

```bash
python predict.py --test-path data/hidden_test.csv --model-path artifacts/model.joblib --output-path predictions/predictions.csv
```

`predict.py` loads the saved bundle, validates that the required feature columns are present in
the test file, reorders columns to match the order used during training, runs inference, checks
that the prediction count matches the input row count and that there are no NaN/infinite values,
and writes a single-column `predictions.csv` (one `prediction` value per row, in the same order as
the input вЂ” `hidden_test.csv` has no ID column to carry through).

## Results

| Model | Features used | CV RMSE (mean В± std) |
|---|---|---|
| Dummy (predict mean) | вЂ” | 28.898 В± 0.075 |
| Linear Regression | all 53 raw features | 28.908 В± 0.074 |
| **Polynomial(degree=2) + Linear Regression** (final model) | `6`, `7` | **в‰€ 0.0000000000 В± 0.0000000000** |

The plain linear model on all 53 raw features performs no better than predicting the mean вЂ”
expected, since the true relationship is quadratic in column `6` and invisible to a model that
only sees linear terms. Once `column_6` and `column_7` are explicitly engineered through
`PolynomialFeatures`, the model recovers the underlying formula almost exactly, with cross-
validated RMSE at the level of floating-point noise (~1e-14).

Fitted coefficients on the polynomial terms `[6, 7, 6ВІ, 6В·7, 7ВІ]`:

```
[ 3.1e-16,  1.0,  1.0,  7.3e-18,  1.1e-16 ]
intercept в‰€ -1.4e-14
```

i.e. the model learned `target в‰€ 0В·(6) + 1В·(7) + 1В·(6ВІ) + 0В·(6В·7) + 0В·(7ВІ) + 0`, which is exactly
the formula identified during EDA.

## Assumptions and design decisions

- **Connectivity / data integrity**: no missing values and no duplicate rows were found in
  `train.csv`; no imputation or deduplication logic was needed.
- **Validation strategy**: plain 5-fold `KFold` with shuffling and a fixed random seed (`42`) was
  used rather than a grouped or time-based split, since there is no stated temporal or grouped
  structure in the data and row order appears arbitrary.
- **Feature selection**: columns `6` and `7` were selected based on EDA evidence (feature
  importance, scatter plots, and an exact-formula check on every training row), not by
  hyperparameter search. The remaining 51 columns (ten ~500-value categorical-like integers, one
  binary flag, and ~40 Gaussian-distributed floats) showed no correlation with the target or with
  the residual after removing `6` and `7`, and were excluded from the final model.
- **Why not stop at the closed-form formula**: `target = col_6**2 + col_7` alone would reproduce
  the result with zero learned parameters, but the submission deliberately builds and validates an
  actual `scikit-learn` model rather than hardcoding the formula, so the pipeline is a genuine
  (if intentionally simple) trained model with a real cross-validation procedure behind it.
- **Random seeds**: `random_state=42` is used everywhere a seed is accepted, for reproducibility.
- **No data leakage**: `predict.py` never has access to the true target; `hidden_test.csv` does
  not contain a `target` column, and no information from the hidden test set was used during
  model selection or training.
